## Title

CI does not build or test the Python repair service

## Description

GitHub Actions runs `pnpm lint`, `typecheck`, `test`, and `build` for the pnpm/Turbo workspaces only. `services/repair` is deployed via Docker on Railway (`services/repair/railway.toml`, `services/repair/Dockerfile`) but has no CI step to `docker build` or run Python tests. Broken `requirements.txt` or Dockerfile changes can reach production without failing PR checks.

## Impact

Regressions in the repair API image or dependencies may ship undetected until deploy or runtime failure.

## Suggested fix

Add a CI job (or step) that builds `services/repair` with Docker and optionally runs `pytest` or `python -m compileall` if tests exist.

**Affected files:** `.github/workflows/ci.yml`, `services/repair/Dockerfile`
