## Title

No single startup validation for all required environment variables

## Description

`.env.example` documents many variables. Some paths validate configuration when used (for example `readDatabaseConfig` in `lib/postgres.ts`, and `proxy.ts` guards API auth), but there is no one central schema (for example Zod or explicit checks at process start) that fails fast in every deploy context if critical vars are missing or malformed.

## Impact

Misconfigured production deploys may partially work or fail late on specific routes instead of at boot with a clear list of missing keys.

## Suggested fix

Add a small env schema module validated once when the server starts (or in `instrumentation.ts` if used), listing required vars per deployment profile.

**Affected files:** `apps/dashboard/lib/env.ts`, `apps/worker/src/index.ts`

- **[code_ref]** Middleware returns 503 when secrets missing only for protected API paths logic.
