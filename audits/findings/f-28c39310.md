## Title

penny_ALLOW_OPEN_API disables dashboard API authentication

## Description

When neither `DASHBOARD_API_SECRET` nor `SUPABASE_JWT_SECRET` is set, production allows all `/api/*` traffic if `penny_ALLOW_OPEN_API` is `1` or `true`. `proxy.ts` logs a warning but still calls `next()`. Attack scenario: operator mistake or compromised env leaves the dashboard API fully open; any client can read or mutate project state exposed by those routes with no shared secret or user session.

## Impact

Unauthorized access to dashboard-backed data and actions (project CRUD, orchestration, imports) depending on route behavior.

## Suggested fix

Remove or gate `penny_ALLOW_OPEN_API` behind explicit local-only hostname; fail closed in production regardless of flag; alert on startup.

**Affected files:** `apps/dashboard/lib/dashboard-secret.ts`, `apps/dashboard/proxy.ts`

- **[code_ref]** Flag enables open API in production when no secrets.
