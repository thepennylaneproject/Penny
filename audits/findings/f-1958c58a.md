## Title

Empty REPAIR_SERVICE_SECRET accepts empty Bearer token

## Description

`repair-callback` compares `Authorization` Bearer text to `Deno.env.get('REPAIR_SERVICE_SECRET')`. If the secret is set to an empty string in deployment, `token !== expectedToken` is false when both are empty, so the handler accepts `Authorization: Bearer ` with no secret. Attack scenario: misconfiguration (empty env in Supabase secrets) leaves the callback open; an attacker invokes the function and updates arbitrary `repair_job_id` rows via the service-role client.

## Impact

Unauthorized updates to repair job status, PR metadata, and orchestration_events linkage.

## Suggested fix

Reject early if `!expectedToken?.trim()` with 503; use timing-safe compare for secrets.

**Affected files:** `supabase/functions/repair-callback/index.ts`

## Proof hooks

- **[code_ref]** Bearer compared to env; no reject when secret missing or empty before compare.
