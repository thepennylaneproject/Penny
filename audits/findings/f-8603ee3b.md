## Title

Orchestration enqueue secret can be read from sessionStorage by page scripts

## Description

Previously, the enqueue secret could be persisted in `sessionStorage`, so same-origin XSS could read it via the Storage API.

**Resolution (2026-04-10):** The live secret lives in **module memory** (`enqueue-secret-store`). `writeEnqueueSecret` does not call `sessionStorage.setItem`. A legacy key may be read **once** to migrate into memory and is then removed. UI copy states “this tab only, in memory.” Residual risk: any XSS in the tab can still attack in-memory secrets—mitigate with CSP and input hygiene.
