## Title

Linear sync API routes reference a table that is never created in migrations

## Description

POST handlers for bulk Linear sync insert into `penny_linear_sync_new` with columns `(project_name, finding_id, linear_issue_id, linear_team_key)` and `ON CONFLICT (project_name, finding_id)`. No migration in `supabase/migrations/` defines `penny_linear_sync_new`. The only related object is `penny_linear_sync` in 20260403105735_penny_manifests_linear.sql, with a different shape (single-row-per-project `state` JSONB, PK `project_name` only). Deployed databases will raise undefined_table errors when these routes run.
