## Title

Captured typecheck artifact is not a real TypeScript compile

## Description

audits/artifacts/_run_/typecheck.txt contains the npm stub message 'This is not the tsc command you are looking for' instead of tsc --noEmit output. The audit batch therefore has no static type-check signal for this run, so type errors could ship without appearing in artifacts.

## Impact

Logic and typing regressions in dashboard/worker/packages may go undetected until runtime or a later CI run.

## Suggested fix

Ensure the audit pipeline runs typecheck from repo root with devDependencies (e.g. npx --no-install tsc or turbo run typecheck after npm ci) and fix PATH/npx so the wrong 'tsc' package is never invoked.

**Affected files:** `audits/setup.sh`, `package.json`

## Proof hooks

- **[artifact_ref]** typecheck.txt shows wrong tsc stub; no project typecheck output.
- **[command]** Root package.json defines turbo typecheck; artifact capture did not invoke project tsc successfully.
