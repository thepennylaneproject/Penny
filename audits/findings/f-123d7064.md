## Title

Worker audit metrics uses explicit any casts around logAuditMetrics

## Description

process-job.ts uses `as any` when passing LLM metric objects to logAuditMetrics (and similar patterns elsewhere per lint). That bypasses compile-time checks: a wrong property name or shape would only fail at runtime or in downstream cost code.

## Impact

Incorrect token or model fields could silently produce wrong cost accounting or failed inserts without a type error at build time.

## Suggested fix

Define a narrow interface matching logAuditMetrics expectations and map llm fields explicitly; remove as any.

**Affected files:** `apps/worker/src/process-job.ts`
