## Title

Deep-link useEffect omits setFilter from dependency array

## Description

eslint react-hooks/exhaustive-deps flags ProjectView: the effect that auto-selects a finding and calls setFilter('all') lists [initialFindingId, findings] but not setFilter. setFilter is recreated each render (not useCallback), so the effect can theoretically see a stale setter reference in edge cases; aligning with exhaustive-deps reduces subtle React 18 strict-mode or future refactors risk.

## Impact

Low probability of stale filter state when combining deep links with session storage filter changes; mostly maintainability and lint compliance.

## Suggested fix

Wrap setFilter in useCallback with stable deps, or add setFilter to the dependency array per ESLint, and verify deep-link + filter behavior in the UI.

**Affected files:** `apps/dashboard/components/ProjectView.tsx`
